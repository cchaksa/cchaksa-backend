# DOMAIN Skill

## 사용 시점
- Context Phase 2 Domain을 수행할 때 참조한다.
- Domain 규칙 수정이나 신규 도메인 모델 정의 시 참조한다.

## 실행 규약
- 순수 Java 객체로 도메인 모델과 서비스 로직을 작성한다.
- 모든 기능은 TDD로 진행하며 예외 포함 시나리오 테스트를 먼저 작성한다.
- 도메인 예외를 정의하고 Application, Infrastructure가 이를 사용하도록 강제한다.
- 값 객체, 엔티티, 정책은 불변성과 명시적 상태 전이를 유지한다.

## TDD 체크리스트
- [ ] 실패하는 도메인 테스트를 작성한다.
- [ ] 최소 구현으로 테스트를 통과시킨다.
- [ ] 리팩터링 후 테스트를 다시 실행한다.
- [ ] `./gradlew test --tests "*Domain*"` 와 전체 테스트를 모두 통과시킨다.

## 금지 사항
- JPA, Spring, Cache, 외부 의존성에 대한 접근을 추가하지 않는다.
- 인프라 세부사항(Repository 구현, RestTemplate 등)을 참조하지 않는다.
- domain 패키지 밖의 클래스에 직접 의존하지 않는다.
