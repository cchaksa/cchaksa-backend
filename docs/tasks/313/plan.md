# 포털 REFRESH 학생 소유권 충돌 방지 구현 계획

## 목표

REFRESH 콜백이 다른 사용자의 학생을 병합하지 않고 현재 사용자의 기존 학생만 안전하게 갱신하도록 한다.

## 작업

- [x] Sentry 최신 이벤트와 현재 트랜잭션 흐름을 문서화한다.
- [x] REFRESH가 사용자 병합을 호출하지 않는 실패 테스트를 추가한다.
- [x] payload 학번 불일치 시 저장 전에 실패하는 테스트를 추가한다.
- [x] 최소 구현으로 REFRESH 병합 제거와 학번 검증을 적용한다.
- [x] 관련 테스트와 전체 테스트를 실행한다.
- [ ] 독립 리뷰 후 `hotfix/313` PR을 `main` 대상으로 올리고 작성자를 Assignee로 지정한다.
