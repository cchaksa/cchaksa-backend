## Decisions
| # | Decision | Reason | Date |
|---|----------|--------|------|
| 1 | 미정의 이수 구분은 canonical 값으로 `기타`를 저장한다 | 데이터 모델 단순화 및 향후 값 추가 시에도 안전하게 처리 | 2026-04-07 |
| 2 | 졸업 요건 API `기타` 영역은 required=0, earned=실제 이수 학점 합계로 노출한다 | 요구사항 테이블을 건드리지 않고도 영역 식별 가능 | 2026-04-07 |
| 3 | 학업 기록 API는 `기타` 과목을 별도 리스트로 제공한다 | 클라이언트가 전공/교양과 분리 표시 가능 | 2026-04-07 |
| 4 | 미지원 이수 구분만 원본 문자열을 `raw_faculty_division_name`에 저장한다 | 향후 `RT`, `교직` 등을 정식 지원할 때 기존 데이터를 식별해 마이그레이션하기 위함 | 2026-05-26 |
| 5 | raw 이수 구분은 `areaType`을 대체하지 않고 학업 기록 API의 `rawAreaType`으로만 추가 노출한다 | 기존 `areaType` canonical 계약을 유지하면서 프론트 표시명을 지원하기 위함 | 2026-05-26 |
| 6 | 졸업 진행 API의 과목 목록에는 raw 원본값을 추가하지 않는다 | 이번 요청의 화면 표시 대상은 `/api/academic/record`의 과목 상세이며 졸업 진행 영역 집계 계약은 유지하기 위함 | 2026-05-26 |

## Risks / Unknowns
- Item: 포털에서 갑작스럽게 다수의 신규 이수 구분을 제공할 수 있음
  - Impact: `기타` 비중이 높아지고, 세부 분석 시 구분 불가
  - Mitigation: 미지원 원본 문자열을 raw 컬럼에 보존하고, 추후 정렬된 목록을 확보한 후 enum 확장 + 데이터 마이그레이션 수행
