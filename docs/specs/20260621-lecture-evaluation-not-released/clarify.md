# Clarify

| No | Decision | Reason | Date |
|----|----------|--------|------|
| 1 | 새 상태값은 `NOT_RELEASED`로 한다 | FE와 BE가 “성적 미공개” 상태를 명시적으로 구분하기 위해 | 2026-06-21 |
| 2 | `PENDING`은 성적 공개 이후 카드/강의평가 미진행 상태로 한정한다 | 기존 PENDING 의미를 명확히 좁히기 위해 | 2026-06-21 |
| 3 | 기존 null row는 수강 내역 기준으로 가능한 범위에서 초기화한다 | 기존 데이터가 FE 상태 판단에서 모호해지는 것을 줄이기 위해 | 2026-06-21 |
| 4 | `SKIPPED`/`COMPLETED`는 동기화로 되돌리지 않는다 | 사용자 선택과 제출 완료 상태를 보존하기 위해 | 2026-06-21 |
