# Clarify

## 결정 사항

- 연동 여부 조회는 학생 프로필이 아니라 사용자 계정 상태 API로 분리한다.
- API 경로는 `GET /api/users/me`로 둔다.
- 응답 필드명은 기존 로그인 응답과 동일하게 `isPortalLinked`를 사용한다.
- 값의 원천은 `User.portalConnected`다.

## 근거

- `/api/student/profile`은 학생 미연결 사용자가 200 응답을 받을 수 없는 구조다.
- 프론트엔드 요구는 학생 상세 정보가 아니라 현재 사용자의 포털 연동 상태 확인이다.
- 로그인 응답에 이미 같은 이름의 필드가 있어 계약 일관성을 유지할 수 있다.
